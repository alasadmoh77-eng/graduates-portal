---

## 📋 نبذة عن المشروع

تُعد **بوابة خدمات الخريجين - جامعة إقليم سبأ** منصة متكاملة لإدارة واستخراج الوثائق والأوراق الثبوتية الرسمية للخريجين إلكترونياً. تهدف المنصة إلى أتمتة وتسهيل جميع الإجراءات الأكاديمية والمالية والإدارية، بدءاً من تقديم الخريج لطلب الوثيقة، ومراجعة السندات المالية، وتدقيق البيانات الأكاديمية، وصولاً إلى التوقيع الرقمي التسلسلي من قبل المسؤولين المختصين وإصدار وثائق PDF موثقة ومزودة برموز التحقق السريع (QR Code). كما تتضمن المنصة بوابة مخصصة لسوق العمل للربط بين الخريجين وأصحاب العمل والشركات.

### الوصف المختصر للأرشيف
> بوابة خدمات الخريجين بجامعة إقليم سبأ هي منصة متكاملة مخصصة لإدارة وتسهيل تقديم طلبات الوثائق والأوراق الثبوتية، والسجلات الأكاديمية، والتوقيع الإلكتروني التسلسلي، والتحقق عبر QR، وتوفير ملتقى لفرص التوظيف والربط مع أصحاب العمل.

## 🎯 المشكلة التي يعالجها المشروع

1. **البيروقراطية والتأخير الزمني:** استغراق التوقيعات اليدوية والمعاملات الورقية أسابيع طويلة لاستخراج وثائق الخريجين.
2. **صعوبة التحقق من صحة الوثائق:** غياب نظام مركزي إلكتروني يُمكن الجهات الخارجية وأصحاب العمل من التأكد الفوري من صحة وموثوقية المؤهلات.
3. **تشتت السجلات الأكاديمية:** الحاجة إلى نظام موحد ومؤتمت يستورد البيانات الأكاديمية من كشوفات Excel وينشئ السجلات وشهادات التقديرات بدقة ودون أخطاء بشرية.
4. **الفجوة بين الخريجين وسوق العمل:** غياب قناة موثوقة تجمع بين خريجي الجامعة والشركات وأصحاب العمل للتقديم على الفرص الوظيفية.

## ✨ المزايا الرئيسية

1. **تقديم ومتابعة طلبات الوثائق:** تقديم الخريج على الوثائق، وتحديد نوع الوثيقة واللغة وغرض التقديم، ومتابعة حالة الطلب لحظياً.
2. **إدارة ورسوم الوثائق والدفع:** إمكانية ربط كل وثيقة برسوم مالية، ورفع الخريج لسند الدفع المالي ومراجعته من الشؤون المالية.
3. **التوقيع الإلكتروني التسلسلي (Sequential E-Signing):** مسار عمل محدد يمرر الوثيقة بالتسلسل على المسؤولين (العميد، المسجل، مدير الخريجين، إلخ) لحين اكتمال جميع التواقيع.
4. **توليد الوثائق الرسمية والـ PDF:** طباعة رسمية عالية الجودة تتضمن الهيدر الشعار والتواقيع المعتمدة ورموز الـ QR Code.
5. **نظام التحقق المباشر من صحة الوثائق:** صفحة عامة تتيح لأي شخص أو مؤسسة مسح رمز QR للتأكد الفوري من بيانات الوثيقة وصحتها في السجل الرسمي.
6. **استيراد السجلات الأكاديمية من Excel:** أداة مخصصة للمسجل الأكاديمي رفع كشوفات Excel وتوزيع البيانات التلقائي على مستويات وفصول ومواد الطالب.
7. **بوابة التوظيف وسوق العمل:** إمكانية تسجيل أصحاب العمل، نشر الفرص الوظيفية، استقبال طلبات الخريجين، واستعراض ملفاتهم.
8. **الدعم الفني والإشعارات:** نظام تذاكر ودعم فني مدمج مع إشعارات داخلية مستمرة للمستخدمين بحالة الطلبات المعاملات.
9. **دعم ثنائي اللغة (عربي / إنكليزي):** إمكانية استخراج الشهادات والسجلات باللغتين العربية والإنجليزية.

## 🏛 المعمارية البرمجية

يتبع المشروع معمارية **MVC متطورة مدعومة بطبقة الخدمات (Service-Oriented Layer)** لضمان فصل المسؤوليات وسهولة الصيانة:

* **Presentation Layer (Blade Views):** واجهات المستخدم التفاعلية المقسمة حسب الأدوار والبوابات.
* **Controllers Layer (`app/Http/Controllers`):** إدارة الطلبات وتوزيع التوجيه والتأكد من الصلاحيات.
* **Service Layer (`app/Services`):** عزل منطق العمليات المعقد في خدمات مخصصة، مثل:
  * `DocumentSigningService`: إدارة مسار وتتابع التوافقية والتوقيع الإلكتروني.
  * `DocumentIssuanceService`: توليد وتأمين الوثائق بصيغة PDF وربطها برمز QR.
  * `AcademicRecordExcelImportService`: تحليل ملفات Excel وتوزيع بيانات الدرجات والمواد.
  * `RequestStatusService`: تتبع تغير حالات الطلب وتسجيل اللوجات المرافقة.
* **Data Access Layer (Eloquent Models):** 27 نموذج برمجياً يعكس الجداول والعلاقات الرياضية بقاعدة البيانات.

## 🌐 البوابات والتطبيقات الفرعية المدمجة

المشروع عبارة عن منصة موحدة تشمل 5 بوابات متكاملة داخل نفس النظام:

1. **بوابة الخريجين (Graduate Portal):** مخصصة لتقديم الطلبات، رفع السندات، وتنزيل الوثائق والتقديم على الفرص.
2. **بوابة الإدارة العامة (Super Admin Portal):** للتحكم بالمستخدمين، الكليات، الرسوم، الإحصائيات الشاملة.
3. **بوابة الشؤون المالية (Finance Admin Portal):** مخصصة لتدقيق سندات الدفع واعتمادها أو إرجاعها.
4. **بوابة المسجل والموقّعين (Academic Registrar & Signers Portal):** تدقيق السجلات الأكاديمية والتوقيع الإلكتروني التسلسلي.
5. **بوابة أصحاب العمل وسوق العمل (Employers Portal):** لنشر وتتبع الفرص الوظيفية والربط مع الكفاءات.

## 🚀 خطوات التثبيت وتشغيل البيئة المحلية

### 1. استنساخ المستودع والدخول للمجلد
```bash
git clone <repository-url>
cd graduates-portal
```

### 2. تثبيت الاعتماديات (Dependencies)
```bash
# تثبيت حزم PHP
composer install

# تثبيت حزم Node.js
npm install
```

### 3. إعداد ملف البيئة
```bash
cp .env.example .env
php artisan key:generate
```

### 4. إعداد قاعدة البيانات وتوافقها
قم بتحديث بيانات الاتصال بقاعدة البيانات في ملف `.env`:
```env
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=graduates_portal
DB_USERNAME=postgres
DB_PASSWORD=your_password
```

ثم قم بتنفيذ الترحيلات والبيانات الأولية:
```bash
php artisan migrate --seed
php artisan storage:link
```

### 5. تشغيل خادم التطبيق وتجميع الأصول
```bash
# تشغيل خادم Laravel
php artisan serve

# في نافذة أخرى، تشغيل Vite للملفات التفاعلية
npm run dev
```
